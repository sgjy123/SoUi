## 支持的参考框架

1. **Ant Design** - 企业级中后台 UI 规范
2. **Material-UI (MUI)** - Google Material Design 风格
3. **Chakra UI** - 现代、可访问性优先的设计系统
4. **Tailwind UI** - 实用优先的 CSS 框架配套组件
5. **不参考特定框架** - 按照 SoUi 设计规范独立实现（推荐）

## 关键原则

1. **必须询问**: 在开始开发前，必须询问用户是否希望参考主流框架
2. **保持 SoUi 风格**: 即使参考了其他框架，也必须保持 SoUi 的设计风格
3. **使用 SoUi 变量**: 必须使用 SoUi 的设计变量和命名规范
4. **文档注明**: 如果参考了框架，必须在文档中添加"参考来源"章节

## 使用流程

1. 用户请求创建新组件
2. 使用 AskUserQuestion 工具询问是否参考主流框架
3. 如果用户选择参考框架：
   - 研究该框架的 API 设计
   - 借鉴合理的 Props 接口和默认值
   - 保持 SoUi 的设计风格
   - 在文档中注明参考来源
4. 如果用户选择不参考：
   - 完全按照 SoUi 的设计规范和现有组件模式进行开发
5. 继续正常的组件开发流程
